# ロードマップ

> 関連: [企画書 §MVP機能 / §将来機能](../hazakura-volume-booster企画書.md) / [ARCHITECTURE](./ARCHITECTURE.md) / [RISKS](./RISKS.md)

Hazakura Boostのリリース計画を、**v0.1（MVP）/ v0.2 / v0.3 / v0.4以降**の4段階で整理します。各段階で「Doneの定義（DoD）」と「受け入れ確認」を明示し、**Doneの定義を満たさない限り次フェーズに進まない**ことを運用上のルールとします。

## 全体スケジュール感

| バージョン | 位置づけ | 状態 |
|---|---|---|
| v0.1 | MVP。**「音を持ち上げられる」ことだけ**を成立させる | public beta PoC |
| v0.2 | 体験改善。出力デバイス切替 / ホットキー / メニューバーに%表示 | 未着手 |
| v0.3 | プリセット。Normal / Video / Lecture / Max | 未着手 |
| v0.4+ | アプリ単位ブースト等。企画書のスコープ外ラインまで含めて再評価 | 未着手 |

> **注意**: バージョンは「リリース番号」ではなく「**機能の到達点**」を表す内部番号として扱います。**App Store / 一般公開用のバージョン番号は配布方針決定後に別途決める**こととし、本ドキュメントのv番号と一致する保証はありません。

## Product Quality Roadmap

v0.1 beta は「音を持ち上げられる」ことを示した段階であり、製品レベルに上げるには機能追加より先に品質ゲートを通す。以下は、今後の実装タスクを選ぶときの優先順とする。

### P0: Safety Gate

ユーザーの耳・出力環境・OS状態を壊さないことを最優先にする。ここを通るまで、外部向けのバイナリ配布はしない。

- [ ] Stop / Quit / `⌘Q` / スリープ前に、必ずゲインが 1.0 へ戻る
- [ ] Activity Monitor からの強制終了後、tap / aggregate device / routing が残らない
- [ ] 二重起動しても音声経路が二重化しない
- [ ] 出力デバイス抜き差し・既定出力変更でクラッシュせず、安全側に倒れる
- [ ] 権限拒否時にクラッシュせず、通常出力または停止状態へ戻る
- [ ] 過大ゲイン・不正設定値・NaN を読み込んでも 1.0 へフォールバックする

### P1: Audio Quality Gate

「聞こえる」から「常用できる」へ上げる。v0.1 beta では完全なマスタリング品質は目指さないが、常時ノイズ・頻繁なブツ音・耐え難い遅延は許容しない。

- [ ] 100%時に原音との差が聴感上ほぼない
- [ ] 200% / 400% で音量差が明確に分かる
- [ ] 10分以上の連続再生で、ブツ音・ノイズ・無音化が頻発しない
- [ ] 体感遅延が用途上許容できる
- [ ] クリッピング時も突然の破裂音にならず、簡易リミッタで抑制される
- [ ] capture underrun / dropped frame / available frames を診断できる

### P2: Runtime Observability Gate

問題が起きたときに、ユーザー報告とDev表示だけで原因候補を絞れる状態にする。Console.app 前提に寄せすぎない。

- [ ] Dev診断に capture buffers / render calls / output gain を表示する
- [ ] underrun count / dropped frames / latest buffer size / available frames を追加する
- [ ] 権限拒否、tap作成失敗、ScreenCaptureKit停止、AVAudioEngine停止をEvent Logへ出す
- [ ] Clear / copy diagnostics 相当の操作を用意する
- [ ] 診断表示は通常利用時に邪魔にならず、必要な時だけONにできる

### P3: Distribution Readiness Gate

手元ビルドから、他人が安全に試せる配布物へ上げる。ここを通るまでは「public beta source / local build」扱いを維持する。

- [ ] Bundle name / bundle identifier / version / copyright を正式化する
- [ ] Developer ID署名済み app を作れる
- [ ] Notarized DMG を作り、Gatekeeperで開ける
- [ ] Privacy Manifest を追加し、録音・保存・外部送信なしの説明と整合させる
- [ ] READMEにインストール、起動、権限許可、アンインストール手順を載せる
- [ ] 配布物に `LICENSE` と必要な third-party notice を同梱する

### P4: UX Polish Gate

機能が増える前に、常駐ユーティリティとしての落ち着きを整える。

- [ ] メニューバー上で ON / OFF / Boost中 が分かる
- [ ] ポップオーバーの状態表示が、停止・権限待ち・エラー・Active を区別する
- [ ] 初回権限説明が短く、マイクではなくシステム音声処理だと分かる
- [ ] キーボード操作とVoiceOverラベルが破綻しない
- [ ] エラー時に「再試行」「停止」「診断を見る」の次アクションが分かる
- [ ] デザインは小さく、静かで、常用時に邪魔にならない

### Release Gates

| Gate | 意味 | 必須条件 |
|---|---|---|
| `beta-ok` | 自分の環境で日常的に試せる | P0の通常終了、安全停止、二重再生抑制が通る。既知リスクをREADMEに明記 |
| `daily-use-ok` | 数日間の常用に耐える | P0完了、P1の長時間再生と音質確認が通る。重大なブツ音・ノイズが残らない |
| `external-beta-ok` | 他人にバイナリを渡して試せる | P0/P1/P2完了、署名済み配布物、インストール/アンインストール手順あり |
| `release-candidate` | 製品版候補 | P0〜P4完了、Notarized DMG、Privacy Manifest、配布物検証、既知リスク一覧更新 |

## v0.1 MVP

### 現在地（public beta PoC）

`spike/core-audio-tap/` に、v0.1 beta として手元利用できる PoC がある。純Core Audio IOProc経路ではなく、Core Audio process tap で元音をミュートし、ScreenCaptureKitで音声を取り込み、ring buffer + AVAudioEngineで出力する構成。

この段階では「広く配布できる署名済みアプリ」ではなく、ソース公開済みの実験ベータとして扱う。Developer ID署名、Notarized DMG、Privacy Manifest、強制終了・スリープ復帰・出力デバイス変更の追加検証は未完了。

### ゴール

> 外部スピーカーのつまみを触らず、メニューバーからシステム音量を 0%〜400% の範囲で持ち上げられる。100%へ即時復帰できる。終了時に安全に元へ戻る。

企画書のMVP機能1〜6を最小構成で成立させ、**Hazakura Boostの存在意義が1つの操作で示せる状態**にします。

### 機能スコープ

- [ ] メニューバー常駐（`MenuBarExtra`、ステータスアイコン表示）
- [ ] クリックでポップオーバーを開く
- [ ] ブーストスライダー（0%〜400%、刻み1%）
- [ ] 「100%に戻す」ボタン
- [ ] ブーストON/OFFトグル
- [ ] 状態表示（「100%」「Boost 180%」「Boost 300%」など）
- [ ] 終了時にgainを1.0へ戻してから停止する安全シーケンス
- [ ] （任意）メニューバーに現在のブースト率を表示

### 明示的にやらないこと

企画書 § やらないことに加え、v0.1では以下も対象外:

- プリセット
- ホットキー
- 出力デバイス切替（デフォルト出力のみ）
- アプリ別ブースト
- 自動起動（launch at login はv0.1ではUIを出さない。実装はしておいてもよい）
- 多言語対応（英語UIで先行し、表示文字列は i18n 前提で書く）
- Sparkle等による自動アップデート機構
- 公証/署名済み配布（Developer ID署名＋Notarizeはv0.2で再評価）

### Doneの定義（DoD）

すべて満たすこと:

1. **機能**
   - [ ] 起動するとメニューバーに常駐する（Dockには出さない）
   - [ ] ポップオーバーのSliderで 0%〜400% を1%刻みで変更でき、音に反映される
   - [ ] 100%ボタンは現在の値に関係なく1.0へ即時戻す
   - [ ] ON/OFFトグルはゲインを1.0へ戻す/戻した状態から復帰する（スライダー値は保持）
   - [ ] **通常終了経路**（メニューからの終了 / `⌘Q` / スリープ前）で、`shutdown()` 経由で **必ず gain が 1.0 へ戻されている** ことがログまたは AudioEngine のステートで確認できる
   - [ ] **強制終了経路**（Activity Monitor からの kill、`SIGKILL`）ではアプリ側 `shutdown()` は走らないことが前提。代わりに、[`docs/TECH_SPIKE.md`](./TECH_SPIKE.md) の PoC で **「OS が tap / aggregate device / routing を自動解放し、極端音量で固着しない」** ことを実機検証する
2. **UI/UX**
   - [ ] ポップオーバーは企画書のワイヤーフレームに準拠
   - [ ] Slider / Button / Toggle がすべてキーボード操作可能（Tab移動、Space/Enterで発火）
   - [ ] VoiceOverで各要素に意味のあるラベルが読み上げられる
   - [ ] ライト/ダーク両モードでテキスト・アイコンが視認できる
3. **品質**
   - [ ] スライダー操作時にメインスレッドが固まらない（60fpsでドラッグに追従）
   - [ ] ポップオーバーを開いたまま5分間放置してもCPU使用率がアイドル時に近い値で推移する
   - [ ] macOS 26 上でクリーンマシンから起動し、初回は `NSAudioCaptureUsageDescription` の権限ダイアログが出る。**許可後、YouTube 等の再生音が通常通り出力され、スライダー操作が音量に反映される**
4. **配布前段**
   - [ ] Developer ID署名なしでローカルビルド・起動できる
   - [ ] クラッシュや致命的なメモリリークが無い（Instrumentsで1時間稼働させたときにリーク0件）
5. **ドキュメント**
   - [ ] `README.md` のステータスが現在の公開段階と一致している
   - [ ] `docs/ARCHITECTURE.md` の不変条件を満たしていることがコードコメントまたはテストで示されている

### 受け入れ確認

- クイックチェック（手動）
  - YouTubeを全画面表示で再生
  - Mac本体ボリュームを50%に固定
  - ポップオーバーで 200% に上げる → 明らかに音量が上がる
  - 100%ボタンで即時復帰
  - ON/OFFトグルでブーストが切れていることが聴感で確認できる
- 受け入れチェックリスト: [ROADMAP §v0.1 受入チェックリスト](#v01-受入チェックリスト) を埋める

### リリース判断

v0.1は「**広く配布しないが、自分で常用できる品質**」をゴールとします。

- Developer ID署名＋Notarize済みDMGを作るかは**v0.1のDoDには含めない**（[PERMISSIONS](./PERMISSIONS.md) 参照）
- v0.1時点で公開する場合はDMGを手動共有し、フィードバックを受ける

## v0.2

### ゴール

> 日常利用の摩擦を減らす。ホットキー1つで100%へ戻したりON/OFFでき、スピーカーを切り替えてもブーストが追従する。

### 機能スコープ

- [ ] **出力デバイス選択**: デフォルト出力以外も選択可能
- [ ] **メニューバーに現在のブースト率を表示**: ステータスラベル機能
- [ ] **ホットキーでON/OFF**: 例: `⌃⌥B`（設定可能）
- [ ] **ホットキーで100%へ戻す**: 例: `⌃⌥0`
- [ ] **スリープ/復帰時の挙動を定義・実装**: スリープ前にgainを1.0へ戻す or 復帰時に復元、いずれかを採用
- [ ] **起動時の自動開始（Launch at Login）**: UIでON/OFF可能
- [ ] **Developer ID署名＋Notarized DMGでの配布経路を整備**
- [ ] **Privacy Manifest（`PrivacyInfo.xcprivacy`）を整備**

### Doneの定義（DoD）

1. 上記機能がすべて動作する
2. **ホットキー実装方式を v0.1 末までに決定**する。実装方式によって追加権限の有無が変わる:
   - **Carbon `RegisterEventHotKey`** 系の実装なら、**アクセシビリティ/入力監視の追加権限なしで済む**場合がある（推奨経路）
   - **CGEvent tap / NSEvent.addGlobalMonitorForEvents** 系の実装なら、**アクセシビリティまたは入力監視のユーザ許可**が必要になる
   - 可能な限り **追加権限が不要な方式を優先**する。必要になったときだけ、初回 1 回だけダイアログを出して以降は鬱陶しくない運用にする
3. Notarized DMGをインストール後、初回起動時にGatekeeperに止められず開ける
4. 起動/スリープ/復帰/出力切替のすべての遷移で、音量暴れが起きない
5. 公式READMEにインストール手順・アンインストール手順が記載されている

## v0.3

### ゴール

> 「よく使う値」を1タップで呼び出せる。プリセット思考で思考停止状態でも使える。

### 機能スコープ

- [ ] プリセット: Normal 100% / Video 160% / Lecture 220% / Max 300%
- [ ] プリセットのカスタマイズ（数値は固定、並び順や名称の変更のみ許可する想定）
- [ ] ポップオーバーにプリセット行を追加
- [ ] 任意: プリセット用アイコン

### Doneの定義（DoD）

1. 4つのプリセットをクリックで適用でき、Sliderの位置/値もそれに同期する
2. 設定は UserDefaults に永続化され、再起動後も復元される
3. ポップオーバーの縦サイズが増えても、マウス/トラックパッドで隠れない

## v0.4以降

企画書の「将来機能」セクションを起点に、**v0.3のフィードバックを踏まえて優先度を再評価**する。

- アプリ単位のブースト
- 特定アプリだけブースト
- 音を出しているアプリ一覧の表示
- 自動ブーストプロファイル
- 高度なリミッタ/クリップ防止
- 簡易EQ
- 設定エクスポート/インポート
- 多言語対応（まずは英/日）

> **注意**: v0.4以降は「**全体ブーストの心地よさ**」を毀損する方向に機能を足さない。アプリ別ミキサー化したい場合は、**別プロダクト（Hazakura Mixer 等）として分岐**させることを検討する。Hazakura Boostは「小さい音を少し大きくする」一点に集中する。

## v0.1 受入チェックリスト

最終リリース前に埋めるチェックリスト。**すべて埋まるまでv0.1完了としない**。

```
起動・常駐
  [ ] LaunchAgent等を経由せず、アプリ単体で起動できる
  [ ] Dockにアイコンが出ない
  [ ] メニューバーに専用アイコンが出る
  [ ] 2重起動してもクラッシュしない（2つ目は何もしない or 既存を前面に出す）

ポップオーバー
  [ ] アイコンクリックで開く/閉じる
  [ ] Sliderで 0%〜400% を1%刻みで動かせる
  [ ] 100%ボタンで即座に1.0へ戻る
  [ ] ON/OFFトグルで音量が変化するが内部値は保持される
  [ ] ESCキーでポップオーバーが閉じる
  [ ] ポップオーバー外クリックで閉じる

状態表示
  [ ] 100%時は「100%」と表示
  [ ] それ以外は「Boost NN%」と表示
  [ ] ON/OFFトグルがOFFのときは「Boost (paused)」のような表示

終了
  [ ] 「終了」メニュー項目から終了できる
  [ ] ⌘Q でも終了できる
  [ ] 終了直前に必ずゲインが 1.0 へ戻る
  [ ] スリープ前にゲインが 1.0 へ戻る（configuredGain は保持）
  [ ] スリープから復帰すると復元する
  [ ] Activity Monitor からの強制終了で AudioEngine が OS 側に tap / routing のゴミを残さない（PoC 検証）

出力デバイス
  [ ] 出力デバイス抜き差しで **クラッシュしない**
  [ ] 必要に応じて Audio Engine を再起動 / ユーザに「Restart required」表示
  [ ] **自動追従は v0.2 スコープ**。v0.1 は「安全側に倒す」だけ

アクセシビリティ
  [ ] VoiceOverですべての主要要素にラベルがある
  [ ] フルキーボード操作で100%ボタンとトグルに到達できる
  [ ] スライダーの現在値がaria的に読み上げられる

品質
  [ ] スライダーを高速に動かしてもUIが詰まらない
  [ ] スリープ⇄復帰を 10 回繰り返しても音量が暴れない
  [ ] **出力デバイスを抜き差ししてもクラッシュせず、安全側に倒れる**（自動追従は v0.2）
  [ ] 1 時間稼働でメモリが膨張しない
```
